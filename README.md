# ML Model Monitoring System

Упрощённая система мониторинга качества модели машинного обучения в режиме реального времени на основе микросервисной архитектуры. Сервисы взаимодействуют через брокер сообщений RabbitMQ (протокол AMQP).

---

## Архитектура системы

```
features ──► очередь X ──► model ──► очередь y_pred ──┐
features ──► очередь y_true ──────────────────────────┤
                                                       ▼
                                                    metric
                                                       │
                                                       ▼
                                               metric_log.csv
                                                       │
                                                       ▼
                                                     plot
                                                       │
                                                       ▼
                                          error_distribution.png
```

## Описание микросервисов

### features

**Файл:** `features/features.py`

Генерирует бесконечный поток входных данных. При запуске загружает датасет `diabetes` из библиотеки `scikit-learn`. В каждой итерации бесконечного цикла:

1. Случайно выбирает одно наблюдение из датасета.
2. Формирует уникальный идентификатор сообщения на основе текущего timestamp.
3. Отправляет вектор признаков в очередь `X`.
4. Отправляет истинный ответ в очередь `y_true`.
5. Делает паузу 10 секунд перед следующей итерацией.

Оба сообщения содержат одинаковый `id`, что позволяет другим сервисам сопоставлять их между собой.

---

### model

**Файл:** `model/model.py`

Сервис предсказаний. При запуске обучает модель линейной регрессии (`LinearRegression` из `scikit-learn`) на полном датасете `diabetes`. Затем переходит в режим ожидания сообщений:

1. Читает сообщение с вектором признаков из очереди `X`.
2. Применяет обученную модель и получает предсказание `y_pred`.
3. Публикует результат в очередь `y_pred`, сохраняя тот же `id`.

---

### metric

**Файл:** `metric/metric.py`

Сервис вычисления ошибок. Одновременно слушает две очереди — `y_true` и `y_pred`. Поскольку сообщения поступают асинхронно, использует буфер в памяти (словарь) для сопоставления:

1. Каждое входящее сообщение сохраняется в буфер по ключу `id`.
2. Как только для одного `id` получены оба значения (`y_true` и `y_pred`):
   - вычисляется абсолютная ошибка: `|y_true - y_pred|`
   - строка записывается в файл `logs/metric_log.csv`
   - запись удаляется из буфера

Формат CSV-файла:

```
id,y_true,y_pred,absolute_error
1669147134.196809,295.0,221.77,73.23
1669147136.824343,153.0,118.44,34.56
```

---

### plot

**Файл:** `plot/plot.py`

Сервис визуализации. Работает в бесконечном цикле с интервалом 15 секунд:

1. Читает файл `logs/metric_log.csv`.
2. Строит гистограмму распределения столбца `absolute_error`.
3. Сохраняет график в файл `logs/error_distribution.png`, перезаписывая предыдущую версию.

## Запуск

### 1. Клонировать репозиторий

```bash
git clone <url-репозитория>
cd microservices
```

### 2. Собрать образы и запустить все сервисы

```bash
docker-compose up --build
```

Docker Compose автоматически:
- поднимет RabbitMQ и дождётся его готовности (healthcheck)
- соберёт и запустит сервисы `features`, `model`, `metric`, `plot`

### 3. Наблюдать за работой системы

Логи всех сервисов выводятся в терминал. Можно фильтровать по конкретному сервису:

```bash
docker-compose logs -f features
docker-compose logs -f metric
```

Файлы результатов появятся в локальной папке `logs/`:

| Файл | Описание |
|------|----------|
| `logs/metric_log.csv` | Обновляется каждые ~10 сек по мере поступления данных |
| `logs/error_distribution.png` | Обновляется каждые 15 сек |

### 4. Открыть интерфейс RabbitMQ (опционально)

Перейти в браузере по адресу: `http://localhost:15672`

Логин: `guest` / Пароль: `guest`

Здесь можно наблюдать за очередями `X`, `y_true`, `y_pred` в реальном времени.

### 5. Остановить систему

```bash
docker-compose down
```
